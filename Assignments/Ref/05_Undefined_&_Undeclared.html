5. What the deference between undefined and undeclare in JavaScript?

What is "Undeclared"?

When a variable is "undeclared", it means that it has not been declared or defined in the current scope.

it means, it hasn't been given a value or a type.

This can happen if you try to use a variable without declaring it first, or if you misspell the variable name.

// This code will throw a ReferenceError because the variable "x" is undeclared
console.log(x);

/* This code will not throw an error because the variable "x" is declared, 
even though it has not been initialized with a value */
let x;
console.log(x); // Output: undefined
                
let x; 
console.log(x); //here x value does not print.
x = 5; // Now, variable is no longer undefined. 
console.log(x); //here x will be print'5'.

What is "Undefined"?

Unlike "undeclared" variables, "undefined" variables have been declared in the current scope, but have not been given a value.

In other words, they have been initialized, but their value is not defined.
